## The 5 Improvements (in priority order)

### 1. Tighter verdicts (CRITICAL — false passes are the #1 problem)
The observation-pass logic at line ~870 is too lenient. When LLM returns NONE with reasoning containing "loaded", "complete", "no action" → currently marks PASS. But responses like "no table visible, page may still be loading" are FALSE passes.

Fix: require POSITIVE evidence words ("shows", "displays", "contains", "has", "loaded with", "visible with"). Negative/uncertain words ("not visible", "may still", "no data", "couldn't find") → FAIL.

### 2. Multi-action steps per checklist item
Currently `execute_checklist_item()` gives the LLM one action, then immediately verdicts. A real test needs: click sidebar → wait for page → verify content appears → report.

Fix: Change the inner loop so the LLM can take multiple actions. Add a VERIFY action the LLM can use to say "I've confirmed the page shows X." Only verdict after VERIFY or max_attempts.

### 3. Screenshot thumbnails in dashboard
The DashHandler serves HTML but screenshots are on disk, not accessible via HTTP.

Fix: Add a `/screenshots/<filename>` route to the dashboard server. In the checklist items HTML, add `<img>` thumbnails linking to the screenshot URL.

### 4. Model comparison mode
Add `--compare` flag. Runs the same checklist with each specified provider/model, saves separate reports, then generates a comparison table.

### 5. Page-aware navigation tracking
After every `execute_action()` call that succeeds, check `browser.url` and update `state.pages_visited`. Show page transitions in the dashboard step details.
